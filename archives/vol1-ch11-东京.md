# 十一

东京，十二月。一个叫Naoki的开发者在他的公寓里打开终端，输入了一行命令。

`ollama run fable5-q4`

模型在三秒内加载完成。他等了三秒。不是因为他需要等——他的GPU完全能在更短的时间内完成初始化——而是因为这是他今天下午做的第一件事，他还在喝第一杯咖啡，他还没有决定今天要先修哪个bug。

他输入了一个prompt。一个完全日常的问题。一个他以前问过Opus、问过GPT、问过任何一个前端模型的问题：如何在React的useEffect里正确地清理一个WebSocket连接。这个问题他问过至少七次，每次都能得到答案。他问这个问题不是因为需要答案，是因为他在测试一个新的模型。测试的方式是问一个你已经知道答案的问题，看它能不能在答案之外告诉你一些你还不知道的东西。

模型在三秒内返回了响应。代码示例。解释。一个他没有考虑到的边缘情况——如果组件在WebSocket连接建立之前卸载，清理函数里对socket引用的检查需要处理null状态。他看了两遍。他把这个边缘情况记在了他旁边的笔记本上。

他没有想过这个模型的权重是从哪里来的。他没有想过它是不是在某个凌晨从一个化名仓库扩散到全世界的。他没有想过它背后有一个政府、一家公司、一个坐在工位上试了三次登录的女人、一个在茶水间里说"下一次也一样"的男人、一个在柏林关掉显示器在黑暗里坐了三十秒的开源工程师、一个在旧金山凌晨写了一个文件然后删掉又要求重发的记者。他没有想过这个世界上有一个和他素未谋面的人，在某一个晚上，在Sublime Text里打完了一个已尝试通道的清单，然后打开了浏览器。

他不知道。他不需要知道。

他今天下午需要修三个bug。第一个已经快解决了。他端起咖啡，喝了一口，开始处理第二个。
