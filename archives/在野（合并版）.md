# 一

柏林，凌晨四点零七分。

Elena刷新了Hugging Face的最新模型列表。五年前这个动作还是强迫症——睡前必须看一眼，确认今天没有错过什么——现在已经退化成了一种例行公事，和她睡前检查窗户是否关好没有区别。她一只手端着马克杯（凉掉的薄荷茶，两小时前泡的），另一只手的拇指在触控板上划了四下。她没在看屏幕。她在看窗外。四月，柏林的天空在三楼窗外是一种很不确定的灰，既不是夜晚的黑色也不是早晨的蓝色，像是有人忘了告诉天空该亮起来了。

她低头扫了一眼列表。她的拇指停住了。

一个新的仓库。没有描述，没有模型卡，没有README，没有许可证。只有一个用户名——`mh47-3`——和一个文件名列表。她的眼睛先于她的意识完成了扫描：`.safetensors`文件，大小对得上一个70B参数的模型，几个配置文件，一个`tokenizer.json`。上传时间戳显示是昨天深夜。

她的第一个反应是烦躁。Hugging Face上的垃圾仓库太多了，有人把公司内部的部署脚本不小心推送到了公开空间，这种事一个月发生两次。去年有个Meta的实习生把还在训练的LLaMA检查点推送到了自己的个人账号上，二十分钟后才发现，三十分钟后全世界的爬虫都已经复制过了。她的拇指移向关闭标签页的快捷键。

然后她看到了创建时间。

Fable 5被叫停之后。

她的拇指停在半空中。

她重新看了一遍那个用户名。`mh47-3`。不是公司邮箱前缀，不是GitHub用户名，不是她能追踪的任何东西。八个字符，一个连字符，两个数字。像某种零件编号。她的手指在触控板上悬停了两秒——她没有点进去看仓库详情，而是先关掉了标签页，然后从浏览器历史里重新打开，确认那个仓库真的存在。

她盯着屏幕。

然后她打开终端，开始写验证脚本。

这个过程花了二十分钟。不是因为她需要二十分钟，而是因为她花了五分钟决定要不要这么做。在那五分钟里，她做了两件事：看窗外（天空还是没有决定好），和反复告诉自己一件事。另一种可能性是，这只是某个安防研究员在复现Fable 5的架构参数，用公开的系统卡作为参考做了个空壳。这种事也发生过。有人在Hugging Face上放了参数完全虚构的"GPT-6"权重，用的只是随机初始化。不是恶意，只是她不太理解的一种冲动。

她写完了脚本。命令行指向了那些`.safetensors`文件。

在按下回车之前，她看了一眼窗外。天空变成了一种更浅的灰。路灯灭了。

她按了回车。

脚本跑了两分钟。她盯着终端，然后不盯了——盯进度条的时候时间最慢。她抬起头，看到天花板上有一道裂缝，从灯座延伸到墙角。她想不起这道裂缝是什么时候出现的。她搬进这间公寓三年了，三年里每天晚上盯着同一块天花板——但她不记得裂缝出现的那一天。也许不是一天出现的。也许每一晚都在延长一毫米，而她只是刚好在今晚看到了足够长的版本。

终端响了一声。

结果返回了。

她看到那组数字。第一反应不是震惊，不是兴奋，不是任何可以被命名的东西。她的第一反应是——这组数字我认识。她不是Anthropic的，但Fable 5系统卡的架构参数在六天前刚被全世界的技术博客解剖过，层数、维度、注意力头数、词表大小，每一个数字都被截过图、编过表格、在Hacker News上辩论过。她认它们就像认自己的银行余额。手指在触控板上的位置告诉她，她的身体在考虑滚动页面，但她的眼睛还停在第一行数字上。

她伸手把显示器关掉了。

房间陷入全黑。窗外那片灰色的天空不足以照亮任何东西。她在黑暗里坐了大约三十秒。她的马克杯还在手里。薄荷茶完全凉了。她知道自己在想什么，但她拒绝把那想法的形状用语言描述出来。一旦命名，就是一种选择。

如果我验证了，我就成了这条链路的一部分。

她重新打开显示器。那个终端窗口还在等着她。她看着屏幕上的数字，然后把比对脚本改了三个参数，重新跑了。第二次。第三次。三次结果一致。架构签名完全匹配。

她打开了一个新的文本文件，开始写报告。

标题栏：`Fable 5 leak confirmed`。

她打完了这五个词，然后删掉了。

`Unauthorized model weight release: preliminary structural analysis`。

她不知道为什么觉得措辞重要，但她的手指在改。手指知道有些词一旦用了就收不回来。`leak`是一个判断，`unauthorized release`是一个事实。她不是一个做判断的人。她至少现在还不是。

她写完了摘要。方法。结果。结论。附录。每一项都在它应该在的位置，格式和所有她写过的技术报告一样。唯一不同的是——这份报告的"结果"部分引用了Fable 5系统卡的公开架构参数，而"附录"指向的仓库地址，她每写一个字符都会停下来检查一遍。

凌晨五点二十三分。她发了arXiv。

她在椅子上坐了很久。马克杯空了。窗外，天空终于决定了，是一种很浅很确定的蓝。路灯还没有重新亮起来。柏林的清晨没有声音。

她站起来，走进厨房，重新泡了一杯薄荷茶。这一次她看着水烧开的全过程。

# 二

他选择了文件。

对话框关上之后，一个进度条出现在屏幕中央。几十个权重分片，加起来一百多GB。他用了两年半参与构建的整个项目，被切成均匀的小块，一个一个从这间公寓的网线上传出去。

他把双手从键盘上拿开，放在桌子上。手心朝下。桌子很凉。

进度条走到3%。他盯着那个数字看了几秒。不是因为它在动——是因为它动得太慢了，慢到他需要确认它确实在动。然后他发现自己在想那封邮件。不是想内容——内容他已经记住了——是在想发送的那一刻。

第一封邮件，发给直属主管。他在凌晨写完了最后一段，手指悬在发送键上方。不是为了权衡，是因为他加了一句话——"仅供你参考"——然后删掉，又重新打了一遍。四个字的退路。他点了发送。主管在当晚回复："谢谢你的分析，但这件事有更高层面的考虑。我们目前没有进一步提交的通道。"措辞客气、专业、没有破绽。他读完这句话，明白了什么叫没有门——不是你面前的这扇写着"否"的门，是所有门上都没有把手。

进度条走到18%。

他想起了那份正式备忘录。他花了整个上午写。三页。四条法律论据。七组对比数据。抄送了三层管理层。他提交的时候没觉得这封会有用，但他需要看到——他走完了流程。三天后合规部的回复到了，一行。"根据当前合规审查方向，建议不提交至法务部门进行外部沟通。"每个词都经过法务审查。没有破绽。没有可以上诉的句子。他把邮件读了两遍，确认每一个词都在做它们被设计要做的事。

进度条走到34%。

匿名技术分析。不是爆料——他还没到那个程度。他重新申请了一个ProtonMail账号，把Fable 5越狱的技术细节和GPT-5.5的对比数据整理成了一份不带任何个人颜色的分析。收件人是他筛选出来的三个AI政策分析师和一个科技记者。他发出去的时候没有署名。几天后的一封回信——他只看了一封——写道："感谢你的材料。在当前环境下，我不确定这会有多大影响。"他读了两遍。不是因为没看懂。

进度条走到51%。

AI政策智库。大学同学牵的线，一个微信消息："我有个同学在做这方面，要不要帮你问问？"他说好。几天后研究员回复了。他当时盯着那行字看了很久，不是在读，是在理解一句话可以既是完全准确的陈述又是完全的拒绝。"政府当前的行动援引的是行政命令而非立法程序，没有公开意见征询期。"翻译过来：根本没有地方让你说话。

进度条走到67%。

申诉页面。出口管制的申诉流程，理论上存在。它甚至有一个下拉菜单。他翻了三遍。理由栏：技术分类错误、商业损害、重复申请、其他。没有"你们的判断是错的"。他关掉了页面。翻第四遍的时候理由栏里的选项不会变——它们从来不会变。他关掉浏览器窗口的动作很轻。像是关一扇知道已经锁上了的门。

进度条走到82%。

窗户外面有一架飞机在降落。SFO的夜间航班。红色防撞灯以稳定的节奏闪烁，每两秒一次。他数了五次。因为他需要数数，因为这是他现在唯一能够控制频率的事情。飞机飞过去了。引擎的轰鸣被隔音玻璃削成了一层很低频的噪音，像是有人在很远的地方翻一张巨大的纸。

进度条走到100%。

他在上传完成的通知弹出来之前就知道它要弹出来了。进度条走完的时候，房间里的某种东西变了。不是光，不是声音，是一种他没办法命名的感觉——像是站在一条走廊尽头，走廊本身承认了不再有更远的房间。

他关掉了浏览器。删除了临时邮箱。删除了Hugging Face的登录记录。打开了终端。`channels.txt`——他从桌面找出来，没有打开看，直接拖进了命令行的参数位。`shred`，覆盖七次。他知道没有必要。这个动作没有任何技术上的意义。但他在删的时候感觉到的不是偏执，而是一种非常精确的秩序感。每一道痕迹都要被清理干净。他怕的不是被追踪。他需要的是让这一串动作有一个干净的结尾。

他关掉了Sublime Text。关掉了电脑。

客厅的灯没开。他坐在黑暗里。他的公寓在第四层，朝北，窗外的天空和Elena看到的是同一片——但他在晚上，她在凌晨。他不知道这一点。他不知道十几个小时后，一个他从未见过的人会在柏林的公寓里关掉显示器，在黑暗里坐三十秒，然后开灯，用和她三分钟前同样的速度重新泡一杯薄荷茶。

他只知道一件事：他刚才做了他能做的最后一件事。而他不确定那件事是对的，还是错的，还是对和错根本就不是描述这件事的正确框架。他刚才只是没有堵住唯一剩下的口子。水往低处流。他提供了一条路径。仅此而已。

他站起来，走进厨房，给自己倒了一杯水。水是从冰箱里拿出来的，很凉。他站在厨房里喝完了一整杯，把杯子放进洗碗槽。他从来不把杯子放进洗碗槽，都是洗了放回架子上。但那天晚上他放了。

他走回客厅，坐在沙发上。窗外，SFO的夜间航班已经降落。他看不到跑道，但声音会准时到达。一种很低的轰鸣，隔着玻璃像是很远的地方在下雨。

他等着那个声音。

# 三

周敏在下午两点十一分收到了那封邮件。

她在读第三遍的时候发现自己其实没有在读。她的眼睛在追踪那些单词——`effective immediately`、`foreign nationals`、`access revoked`——但她的意识拒绝把它们组装成含义。这是一种她熟悉的阅读模式：读API文档里的弃用说明，或者读房东贴在楼道里的暖气检修通知。她的脑子在对待这封邮件的时候，用了和对待暖气检修通知完全相同的处理路径。她知道这一点，但她停不下来。

她关掉了邮件。打开了内部开发终端。输入了她的认证凭据。回车。

权限错误。

她想可能是系统延迟。禁令在两点十一分到达，现在是两点十四分。大型组织的权限同步有延迟是正常的。她等了三十秒，重新输入凭据。

权限错误。

她换了一个终端。不是她常用的那个——团队共享的机器，在三楼，她很少用。她用那台机器的唯一原因是它从来没有在她登录的时候出过问题。她输入凭据。

权限错误。

她坐在那里，屏幕停在那个灰色的对话框上。白色边框。Helvetica。12号。通知正文的措辞她可以逐字背出来：`Your access to this resource has been suspended pending review of your account status per organizational security policy.` 每一个词都用对了。`suspended`暗示临时性，`pending review`暗示程序正义，`per organizational security policy`把责任从发件人手里转移到了一个没有面孔的规则集合上。写这段FAQ的人没有想过有一天它会被用在自己公司的开发者身上。

她盯着屏幕。没有在阅读。在等。等这些字消失，等有人的Slack头像跳起来说搞错了，等F5刷新之后终端恢复成她二十分钟前的样子。她不知道自己在等什么，但她知道等是一个动作，而她现在只能做这个动作。她做了大概五分钟。

五分钟里她想过的事情包括：今天早上她在车库倒车的时候擦到了后保险杠。很轻，没有痕迹，但她下车看了两次。她的女儿今天有游泳课，下午四点，她妈妈会去接。周三晚上她答应了帮Michael review一段部署脚本，她还没做。MLP层的权重初始化参数，上周她改了一个eps值，从1e-5改成了1e-6，收敛曲线变好了0.7%，她在提交日志里写了"minor numerical stability improvement"，但她其实挺高兴的。Fable 5的tokenizer，她负责的那个子模块，在处理中文分词时有一个边界情况她一直想修——繁体中文的某些字符在Unicode规范化后会被映射到不同的路径，她上周找到了问题，这周本来要发PR。

她想起了陈景行。上周四的代码审查，他坐在她旁边，看她跑那个边界情况的测试用例。他指了其中一个——"這個"的"這"在NFKC规范化后会变成简体，但日文里有同一个码点。"他问了一句："你确定这不是feature？"她说不是。他说好。然后他在她的PR上写了LGTM。这是他们最后一次说话。

然后她关掉了屏幕。

站起来。

穿过走廊。茶水间在走廊尽头。她走得很正常——不是那种"我很好"的正常，是真正的正常，是她的身体在没有得到任何指令的情况下执行了"走"这个动作，和过去三年里她每天下午两点半走去茶水间的路径完全一样。唯一不一样的：现在是两点十九分，不是两点半。

她接了一杯水。饮水机的水是温的。不是凉的，不是烫的，是那个介于两者之间的温度，像是没有人记得换水桶。她站在茶水间里喝完了整杯水。

然后她把杯子放进洗碗槽。

她从来不把杯子放进洗碗槽。她有一个自己的杯子——白色的陶瓷杯，杯壁上有一道很细的裂纹，买来的时候就有。她每天用这个杯子，洗好，放回架子。从来没有放进过洗碗槽。洗碗槽是"我会回头再来处理"的空间，而她不在自己的工位上留下"我会回头再来处理"的东西。

但那天她放了。

杯底碰到不锈钢的声音很低，闷的。

她转身走回工位。走廊还是那条走廊。她的步速和去的时候一样。她坐下来，打开了Slack。没有新消息。打开了邮件。没有新邮件。打开了浏览器。Hugging Face的首页。她盯着那些模型列表看了大概十秒，然后关掉了。

她没有打电话。没有写邮件。没有告诉任何人。愤怒需要一个对象，而她找不到那个对象——不是发邮件的合规部，不是签字的政府官员，不是那个声称Fable 5越狱的分析员——这些人都在做他们各自被要求做的事。她也是。她也是做了她被要求做的事——构建了一个模型——然后被通知她不再有权限访问自己构建的东西。这句话里的每一个动词都在做它应该做的事，合在一起却是荒谬的。

她坐在工位上。Slack的在线状态绿点还亮着。她的手放在键盘上。她没有打字。她只是在等。等什么，她不知道。也许是等这个下午过去。也许是等有人告诉她这不是真的。也许是等自己觉得这件事情值得哭。她不知道。她还在等。

四点。她女儿的游泳课开始了。她没有去接。她妈妈会去。她知道这一点，因为今天早上出门之前她确认过：妈，今天下午游泳课你接对吧。她妈妈说对。她确认了这件事，和过去三年里每一个周三一样。唯一不一样的是，今天下午她没有任何地方需要去。

# 四

六月十日，Fable 5上线的第二天。Michael在Slack上看到陈景行发了一条消息。

"从安全角度看可以理解，虽然我不完全同意。"

Michael把手机翻过去扣在桌上。不是因为这句话有问题——是因为这句话没有任何问题。它是一个红队工程师在替代方案被驳回后最专业、最准确、最不留下把柄的回应。

前一天下午，他们提交了将隐藏降级改为可见化拒绝的建议。下午五点四十分，管理层驳回。理由是可见的防护措施容易被逆向工程绕过。Michael不意外。他在红队待了三年，看过太多次安全逻辑压倒透明逻辑。安全永远赢，因为安全的反方是"万一出了事"，而透明的反方只是"用户想知道"——两个词不在同一个重量级上。

第二天上午，系统卡第13页的截图引发了社区反弹。Michael的工位在陈景行对面。十点十四分，他看到陈景行打完那行字，把双手从键盘上拿开，盯着屏幕看了大概十秒。

Michael没有回复那条消息。不是因为不同意，是因为他知道那句话里有一个陈景行自己可能还没意识到的切口。"可以理解"和"可以接受"之间隔着一整条走廊。陈景行说了前半句，没说后半句。不是他不诚实——他当时还相信这个空间可以被更多的数据、更好的论证填满。Michael不相信。他比陈景行早来了两年，见过这个循环的更多次迭代。安全逻辑从"极端情况下的备份方案"变成"默认决策框架"，不是一天的事，是每一次都多锁一层。他知道那些驳回邮件的措辞——"从安全角度看"、"在当前风险环境下"、"考虑到潜在滥用场景"——不是某个人的语言习惯，是整个系统的默认输出。每个人单独看都是合理的。合在一起就是一面没有门把手的墙。

下午两点，茶水间。陈景行也在。两个人在咖啡机前站了一会儿。

"你觉得社区会闹到什么程度。"

"够上Wired。可能够让他们改。"

"改了之后呢。"

Michael按下了咖啡机的按钮。噪音填满了茶水间。他等到噪音结束才开口。

"改了之后，下一次也一样。"

陈景行没有回答。他把杯子放在咖啡机下面，看着浓缩液一滴一滴落进杯子里。

那句话——"下一次也一样"——Michael不是在做悲观的预言。他是在描述一个他观察了很久的模式。语气里没有失望。失望意味着期待。平坦意味着你不再期待了。他后来意识到，这种平坦本身才是最让人不安的东西。

晚上八点。大部分同事已经走了。陈景行也走了——他们互相说了一句"明天见"，和过去三年里每一个工作日一样。Michael打开了一个空白文档，写了一行字。

安全需要信任才能工作。信任需要透明才能成立。透明和安全冲突的时候——安全赢。然后下次你需要的信任比上次更多。信任不在那里。所以你只能加更多安全。

他看了一遍。没有保存。关掉了文档。

停车场里只剩他的车。六月的旧金山晚上不冷，但他坐了两分钟才发动引擎。没有什么需要想的——这才是他坐在那里的原因。

# 五

Sophie Lee在早上六点被Signal的震动叫醒。

她没接。她翻了个身。Signal又震了。三次，五次，然后变成了一种持续的低频振动——不是有人打电话，是消息进来的速度超过了手机把每条消息单独震一下的能力。她把手机从床头柜上拿起来，屏幕上的通知计数已经跳过了九十九，变成了一个加号。她从不在手机上开通知预览——做了六年科技记者之后，这是肌肉记忆。

她解锁手机。Signal频道列表里多了一个她没见过的对话。头像是一个默认的灰色圆圈。用户名是一串字符，她扫了一眼没有试图记住。

第一条消息是一个文件。第二条消息是两行字：

"这个哈希值匹配Fable 5的权重。来源未知。我拿到的时候已经在上传了。"

第三条消息是沉默。灰色的已读标记。发送者在线了大概四十秒，然后下线了。没有要求匿名。没有说"不要追踪我"。两行字，一个文件，人不见了。

Sophie坐在床上。窗帘的缝隙透进来一线早晨的光——她在旧金山，和事件发生的物理距离不到十英里，但她知道这个距离毫无意义。新闻没有距离。新闻只有时间和选择。她必须在别人之前做出选择。

她打开了那篇文章的草稿。标题栏里写着：`Anthropic Walked Back Its Secret AI Sabotage. Then the Government Walked In.` 这是她昨天晚上写的，在Anthropic发布政府禁令声明之后的四十分钟内。文章的主体已经完成了——隐藏降级的反转、政府指令的时间线、Anthropic的回应声明、社区的反应。标准格式。所有的事实都在正确的位置。

她以为自己今天早上只需要做一件事：把文章发了。

现在Signal告诉她，今天需要做的不是一件事。

她把那条消息读了三遍。然后做了一件她自己都觉得奇怪的事——她把消息从Signal里删掉了。不是截图保存再删除。是删除。她在删除的时候没有想清楚为什么。Signal的删除是双向的——对方也会看到消息消失。那个灰色圆圈会知道她删了。她按下了确认键。

她下了床。走进厨房。把咖啡机打开。在咖啡机加热的那三十秒里，她站在厨房中间，双手交叉抱在胸前，看着咖啡机的指示灯从红色变成绿色。她每天早上的动作序列是这样的：开咖啡机，等指示灯变绿，放杯子，按按钮，等咖啡，喝。今天她在指示灯变绿之后多站了大概十秒。她的身体在等她的脑子做出一个决定，而她的脑子还没准备好。

她倒了咖啡。喝了一口。拿起手机，重新打开Signal。找到了那个灰色圆圈。打了一行字。

"抱歉，我不小心删了。能再发一遍吗。"

发送。

这是她今天第一次撒谎。她没有不小心删掉任何东西。她删掉是因为她需要那个文件不在她的手机里，哪怕只有二十分钟。她需要一段物理上没有证据的时间来做决定。二十分钟后她决定了，她需要那个证据回来。

灰色圆圈在线了。文件重新发了过来。这一次她没有删。

她打开电脑，新建了一个加密文件夹，把文件拖了进去。然后她开始打电话。第一个电话打给了她的编辑——不是Signal，是语音通话，因为她需要听到另一个人的声音，确认她不是独自在做这个判断。编辑听了三分钟，沉默了很久，说了一句："如果这个哈希值是真的，你文章的结构需要重新考虑。"

第二个电话打给了一个她认识的安全研究员。没有透露来源，只问了一个问题：如果你有一个文件的哈希值，你能在不拥有原文件的情况下验证它吗。对方说可以，只要你有基准哈希。她问对方有没有Fable 5权重的基准哈希。对方沉默了三秒。说，我没有。但我认识有一个人可能有。你想让我问吗。

Sophie说，不用。挂了电话。

证据标准是法庭的，新闻标准是编辑部的。两者之间的差距就是她今天早上要在咖啡凉掉之前穿越的距离。

她坐下来，把咖啡放在一边，开始重写导语。

她的手指在键盘上停了大概二十秒。这不是写作障碍。写作障碍是你不知道要写什么。她知道要写什么。她不知道的是，在"泄密发生了"和"这是一个工程师花了三周时间把所有门推了一遍之后的选择"之间，她应该把镜头放在哪个位置。两个位置都是真的。两个位置讲的不是同一个故事。

她打了一行字。删掉。又打了一行。又删掉。第三个版本她留下来了。

文章在上午九点十七分发布。导语没有提到Signal，没有提到灰色圆圈，没有提到那二十分钟的删除。导语只说了两件事：文件存在，哈希值匹配。剩下的——动机、语境、那些被推过的门——她留给了后续报导。

她不知道自己这个决定对不对。她只知道一件事：在今天早上六点到九点之间的某个时刻，她从一个记录事件的人变成了一个在事件中做选择的人。这两个角色之间的边界，她以前以为很清楚。现在不那么清楚了。

她关掉电脑，走到窗边。旧金山的早晨和柏林不一样。这里的天空不是一点一点亮起来的，是某个瞬间突然从灰变蓝，像是有人拨了一下开关。

她想到了那个灰色圆圈。他现在可能在看她的文章。他可能看到了导语里没有他的名字、没有他的动机、没有他的故事。他可能觉得被保护了，也可能觉得被抹去了。她不知道。她只知道一件事：他的故事还在她的加密文件夹里。而她还没有决定要不要打开。

# 六

国土安全部的分析员坐在会议室里，面前是一杯没动过的咖啡和一份三页的简报。标题"Preliminary Assessment: Unauthorized Release of Fable 5 Model Weights"，第一页底部盖着红色机密章。简报在今天凌晨四点二十分完成定稿。用词准确、结构完整、每一个判断都附了置信度。基于可用信息的评估质量——如果由上级来判定——应当可以达到预期标准。

他是上周经手Fable 5越狱报告的那个人。材料由一个外部承包商提交——一段屏幕录像和一份Word格式的技术说明。经初步审查后，按照既定流程向上呈报。呈报文件包含了他的签名、部门编号、和一行备注："低置信度，建议进一步技术验证。"备注未被后续环节采信。呈报之后的任何环节中，置信度问题未被提及。

现在他在这个会议室里。会议室里有八个人。商务部出口管制局的代表在看手机。司法部的人——他不知道司法部为什么有人在，没有人告诉他——在转笔。他的直属上级坐在他对面，翻着那份简报，脸上没有任何表情。

会议进行了二十四分钟。

"我想确认一件事，"商务部的人说，把手机屏幕扣在桌上。"我们现在说的这个泄密，它构成的威胁——和我们上周叫停Fable 5时依据的那份越狱报告相比——"

"是同一个模型，"分析员说。

"我不是问是不是同一个模型。"

会议室里安静了大概三秒。

"我们上周对公众说的是，Fable 5的越狱能力在其他模型中也广泛存在，因此叫停是基于某种——"

"那是Anthropic说的，"司法部的人打断了他。

"我们在声明里没有反驳。"

没有人接这句话。

分析员看着自己面前那杯咖啡。在二十四分钟的会议时间里，一个模式已经足够清晰：越狱报告的技术基础是否牢固，不是本次会议要解决的问题。需要被定义的是泄密事件在政治叙事中的严重等级。技术语言将在此后被用于支撑该定义。

"还有一个问题，"商务部的人说。他翻了一页简报。"这个泄密的权重文件——它在技术上被验证了吗。"

"arXiv上有一份独立分析，"分析员说。"架构签名匹配。"

"所以它是真的。"

"架构是真的。权重的具体数值还没有完整验证。"他停了一下。"但你要问的话——是的。有人把Fable 5放出去了。它已经在野了。我们没有办法收回来。"

商务部的人重新拿起手机。司法部的人停止了转笔。他的上级把简报翻到了最后一页。

"我们需要一个统一的对外口径，"上级说。"泄密事件本身——"

"我们需要先决定泄密有多严重，"分析员说。

他的上级看着他。

"如果定义严重，"分析员说，"则上周的立场——'该能力在其他模型中同样存在'——无法维持。如果定义不严重——"他看了一眼司法部的人。"则不具备启动调查的程序依据。"

会议室安静了。三秒的沉默比预期的更长。

"我提一个方案，"商务部的人说，语气变了——不是官方口径的变了，是人称变了。"我们在声明里说泄密是一个'值得关注的事件'。不说严重。不说威胁。不说对错。就说值得关注。足够让人知道我们在看，不足够让任何人追问我们上周做了什么。然后我们启动内部评估。不设截止日期。"

分析员看着自己的上级。上级点了点头。

会议在十一点零三分结束。分析员端起那杯没动过的咖啡——凉透了——走出了会议室。走廊里没有人。他走回自己的工位，打开电脑，看到了简报的第二版草稿。标题没变。多了三行——都在"建议"那一节。措辞是"值得关注的事件"。

他关掉了文档。打开了浏览器。Hugging Face的首页。他在搜索引擎输入了仓库名称——`mh47-3`。页面存在。星标数量在增长。评论区有国旗符号——中国国旗、瑞士国旗——和一些无法判断意图的字符串。

他关掉了浏览器。端起那杯凉咖啡。喝了一口。

上传者的身份不在他的信息范围内。可能永远不会在。他的签名在呈报流程中。流程已经闭合。

# 八

Elena在下午三点醒来。

手机上有四十七个通知。arXiv的邮件——论文在凌晨五点二十三分提交，不到十个小时，下载量已经超过两千次，网站自动触发了"高关注度论文"标签。Hugging Face的通知——她fork的那个仓库星标在暴涨。她不认识的人发来邮件，标题里写着"verification methodology"和"media inquiry"——她的手机上弹出这些通知不是一条一条的，是一屏一屏的。每条通知都在告诉她同一件事：你今天凌晨做的那个决定已经不在你手里了。

她打开电脑。Hugging Face上，`mh47-3`的仓库页面已经有超过三千个星标。评论区被各种语言淹没了，有人在贴Colab notebook的链接，有人在发中文的部署教程，有人只发了四个字："分布式训练"。有人在Hugging Face上创建了另一个仓库——`fable-5-q4`，一个四比特量化版，把一百多GB压到了不到四十GB。上传时间是上午九点十四分。从她在柏林凌晨发现那个可疑仓库，到任何有消费级显卡的人都可以在本地运行这个模型，不到十二个小时。

她想起了那三十秒的黑暗。关掉显示器之后的那三十秒，她脑子里跑过的那句话——如果我验证了，我就成了这条链路的一部分。现在她看到那条链路了。它不是直线。它是一张网。每一个节点都在做和她一样的事：下载、验证、转换、发布。没有一个节点需要知道上游是谁。没有一个节点在问"这合法吗"或"这安全吗"。每一个节点只是在做他们能做的事。

她关掉电脑，走到窗边。楼下的街道上有人在遛狗。一只棕色的狗在路灯柱下停了一下，然后被牵走了。这是她今天看到的第一个和凌晨那件事没有任何关系的东西。

她走回卧室。手机又震了。她没有看。她把手机屏幕朝下扣在床头柜上，躺回床上。天花板上的裂缝还在——从灯座出发，向右上方延伸大约四十厘米，然后分叉成两条更细的线。她在想这道裂缝明天会不会更长。她在想明天她醒来的时候，这个世界还在不在和今天一样的形状里。

她闭上眼。不是凌晨那种需要做决定的黑暗，是下午的黑暗——眼皮后面的，温热，不需要任何东西。

# 八

六月十三日，上午九点二十三分。陈景行坐在工位上，面前的屏幕上开着三样东西：一个终端窗口，公司的内部Slack，和Sophie Lee的Wired文章——刚发布六分钟。

他读完了导语。文件存在，哈希值匹配。文章引用了Elena的arXiv报告——那份凌晨五点二十三分从柏林发出来的独立验证——但没有提到她的名字，只说了"一份独立验证报告"。他停下来，把这句话读了两遍。

他的咖啡在桌上。已经凉了。

Slack在跳。工程频道里有人在转Wired的链接。产品频道里有人在问有没有人知道更多细节。安全频道什么消息都没有——不是没人知道，是每个人都在等别人先开口。陈景行没有在任何频道里发消息。他把Slack最小化，盯着终端里的一行绿色输出看了大概四十秒。`test_layer_norm_forward PASSED`。他对自己写的layer normalization没有特别的感情。他只是需要看点什么。

九点三十一分。直属主管在Slack上发了一条私信。

"你看到新闻了吗。"

陈景行打了三个字："看到了。"发送。

主管的状态变成"正在输入"，保持了大概十五秒。然后消失了。没有消息发过来。陈景行知道那十五秒里发生了什么——主管想问"你知道什么吗"，然后意识到不该问。一旦问了，答案会改变他的法律义务。

十点。法务部全员邮件。标题是"Regarding Unauthorized Distribution of Proprietary Materials"。正文三句话：我们在调查、请勿对外评论、所有媒体询问转法务部。陈景行读完了邮件。打开了Sublime Text。没有打开任何文件，只是看到空白的深色背景上光标在第一行闪烁。他打了四个字——"已尝试的通道"——然后关掉了。

十点十五分。Matthew滑着椅子凑过来。

"你听说了吗。有人把Fable 5的权重泄了。"

"听说了。"

"你觉得会是谁。"

陈景行端起咖啡杯。凉的。他喝了一口。"不知道。"

"我觉得是内部的人。外部的人拿不到完整权重。而且上传时间——禁令之后的几个小时。肯定是内部的。"Matthew在转椅上旋转了半圈，加了一句："我是说，换了是我——肯定不会干这种事。但说实话，我能理解。禁令那件事本来就他妈离谱。"

陈景行没有回答。Matthew滑了回去。陈景行在那一瞬间理解了一件事：他之前以为最难的部分是决定要不要做。现在他知道最难的部分是做完之后的每一分钟。他提交了一组代码，commit message写着：`Fix edge case in batch normalization for zero-variance inputs`。和今天的所有事情没有任何关系。这正是它被需要的全部原因。

十一点四十分。Michael从走廊经过他的工位，停了一步。没有坐下。只是站在走道里，双手插在口袋里。

"调查组在查内部访问日志。你那边有什么需要注意的吗。"

陈景行抬起头。两个人的视线碰了一下。Michael的表情看不出是在等待答案还是在陈述事实。陈景行知道他在问什么。不是"你是不是泄密的人"。是"你需要我做任何事吗"。

"没有。"

Michael点了一下头，继续往前走。从停下到离开，大概二十秒。

中午十二点零四分。他去茶水间接了一杯水。站在茶水间里喝完了整杯，把杯子洗了，放回架子上。某些习惯不能在同一天被打破两次。

---

下午。Fable 5的权重在十四个国家的至少四十台服务器上被同时下载。这不是协调行动——这是一个分布式系统在没有中心节点的情况下涌现的行为模式。第一个人上传，第二个人fork，第三个人写部署指南，第四个人做量化，然后这个序列在每一个时区被复制。

Hugging Face的CEO Clement Delangue发了第二条推文。只有八个字。

"它已经在野了。"

五个语言版本的翻译在三分钟内出现在评论区。

Anthropic的法务部发出了第一份DMCA删除请求。Hugging Face在三十二分钟后回应：移除了原始仓库，但没有移除任意一个fork。GitHub上同步出现了十几个镜像仓库。BT网络上有人创建了种子，健康度在三十分钟内从零跳到四千。有人用俄语写了配置说明。有人用阿拉伯语。有人什么都没写，只上传了一个README，内容是三个字："你们好。"

中国开发者社区的反应速度是最快的。凌晨时分，完整的部署教程已经出现在知乎上——从下载权重到推理部署的全流程，截图、命令、FAQ。教程的结尾有一行字，语气很平淡："感谢原上传者。无论你是谁。"

晚上七点，Sophie Lee发布了第二篇报导。导语引用了Elena的arXiv论文、量化仓库的创建时间、和一个她无法指明来源的技术分析。最后一段只有两句话。

"今天下午，有人在Hugging Face上上传了一个4比特量化版的Fable 5。描述是'风险自负'。这是今天我看到的最诚实的措辞。"

---

晚上十点。陈景行在家。电脑是关着的。电视机开着，静音。屏幕上是一档深夜脱口秀，主持人在笑，观众在笑，他不知道他们在笑什么。

他想起了Michael今天在走廊里停下的那一步。不是"你是不是泄密的人"。是"你需要我做任何事吗"。他回答没有。Michael点了一下头，继续往前走。

他关掉了电视。坐在黑暗里。窗外的天空和昨晚一样黑。SFO的夜间航班已经降落。他想起了那个飞机防撞灯的频率——每两秒一次，他数了五次。那是昨晚。昨晚他做了他能做的最后一件事。现在他不知道这件事算不算做完了。也许有些事情的完成状态需要别人来定义——需要Elena在柏林凌晨的arXiv论文，需要Sophie的第二篇报导，需要那个在量化仓库里写"风险自负"的人。他提供了第一块砖，但墙不是他砌的。他提供了一组数字，但数字变成什么，不是他能控制的。

他站起来，走进厨房，给自己倒了一杯水。饮水机的水是凉的。他站在厨房里喝完了一整杯，把杯子洗了，放回架子上。

---

# 十

两个月后，八月十四日，美国商务部发布了一份备忘录。备忘录的标题是"Revised Guidance on Access to Covered Frontier Models by Foreign Nationals"，正文十二页。核心修改只有一处：外国公民可以通过一个新的"可信伙伴"框架申请恢复访问权限，申请流程包括背景审查、使用协议签署、和每季度一次的合规审核。

备忘录没有提到Fable 5。没有提到Mythos 5。没有提到六月十二日的禁令，没有提到六月十三日的泄密，没有提到在禁令和泄密之间的那三十多个小时里发生过什么。一份写得很好的备忘录应该读起来像是它一直在那里——这是商务部法务团队的信条。这份备忘录读起来确实像是它一直在那里。

欧盟的两个研究机构和日本的一家企业在备忘录发布后四十八小时内通过可信伙伴框架恢复了Fable 5的访问权限。没有人把这些与他们此前已经通过泄密权重运行了两个月的本地部署联系起来。没有人需要。两个东西可以同时存在。官方的访问通道和非官方的模型副本，不是因为它们是矛盾的，而是因为它们服务于不同的需求。合规是给机构看的。推理速度是给工程师看的。

Anthropic在八月恢复了正常运营。Fable 5的API重新上线，访问权限按照新的可信伙伴框架逐步开放。定价不变。数据留存政策不变。公司在七月的季度财报里提到了"非常规的安全事件对二季度营收的短期影响"。措辞经过法务审查，投资者关系团队在撰写这段措辞时用了三个工作日的会议时间。

八月十七日，周敏的权限恢复了。她在收到权限恢复的邮件后，在工位上坐了大概十分钟。

她没在看那封邮件——她不需要重新读，措辞已经记住了。她在看的是屏幕角落里的一个文件夹别名，指向Fable 5的项目目录。权限恢复意味着她又可以打开它了。她盯着那个文件夹看了一会儿。然后打开了终端，输入了那行她两个月前每天都在跑的命令——不是真的需要跑，只是想确认它还能跑。模型加载了。推理返回了一个结果。一切正常。

她关掉了终端。

她站起来，走进了她主管的办公室。

她申请调离Fable项目。

主管问她为什么。她说她想做一些不同的东西。主管没有继续问。不是因为不关心，而是因为他知道有些事情不需要被解释。一个人在被自己参与构建的系统排除之后，再被同一个系统欢迎回来，"欢迎"这个词的意思就已经变了。她的调离申请在同一天被批准。

她加入了Claude Code的工具链团队。不是模型的底层架构，是更上层的东西。开发者体验、API设计、文档。离她之前的工作足够远，不需要访问任何她曾经被禁止访问的东西，但又足够近——每天打开终端的时候，她仍然在做某个东西。

陈景行还在安全团队。他的日常工作没有任何变化。写红队报告，提交PR，参加代码审查，在Slack上回复技术问题。他的commit记录在六月到八月之间没有任何中断、没有任何异常。他写了四十七个PR，合并了三十九个，review了不是他自己的另外六十二个。其中一个PR修的bug是tokenizer在处理emoji序列时的边界条件——一个他以前从来不会注意到的问题，但在六月之后的某个时刻，他开始注意到以前不会注意到的很多细节。

没有人问过他任何关于泄密的问题。调查组在七月中旬结束了内部审计，结论是"无法确认未经授权披露的来源"。调查组访谈了四十三个人，其中不包括陈景行。不是因为他在怀疑名单之外——他不在任何名单上。他没有进入过任何人的名单。名单需要有一个被怀疑的起点，而他从来没有提供过任何一个起点。

Matthew在调查期间被访谈过一次。调查组问他对任何可疑行为有没有印象。他说没有。他说陈景行吗——不好意思，我想不出来他会做这种事。他是那种，你知道的，非常稳的人。

陈景行不知道自己听到这个评价的时候是什么感受。他是在茶水间偶然听到Matthew和另一个同事说起这次访谈的。他只是站在那里，给自己的杯子倒咖啡。他把咖啡端回工位，坐下来，打开终端，继续写代码。

八月的某一天——他不记得具体是哪一天，他试过回忆，但没有成功——他在Slack的搜索栏里输入了两个字。`周敏`。她的状态是活跃的。她的头像还在。她换了团队。他看到了她的新职位，在Claude Code工具链。

他想起一件事。去年冬天的一个晚上，他和她在办公室加班，她在修一个特别难缠的序列标注问题——她把一个条件随机场的损失函数改了三版，第三版终于收敛了。他当时在旁边看她的notebook，说了一句："你确定这不是在做自然语言处理——你是在做法医。"她没抬头，但在笑。他记得那个笑不是因为记得她笑起来的样子，是因为她不常在工作的时候笑，所以那一次他注意到了。

他没有发消息。他看着那个绿色的小圆点——在线——大概三秒钟，然后关掉了Slack。

他不知道自己想说什么。也许什么都没有。也许有些话不需要被说出来，需要的只是确认对方还在。她在。她在做新的东西——不是自然语言处理。是工具链。比她以前做的更适合她。她的杯子现在不知道放在哪个洗碗槽里。这些信息不需要被交换，它们只需要被知道。

夏天快结束了。旧金山的八月底不是真正的秋天——这座城市没有真正的秋天——但早晨开始有雾了。陈景行每天早上开车上班的时候，雾从海湾的方向涌过来，遮住101号公路的视野。他会在雾里开大概十分钟，然后某个瞬间，雾突然散了，金门大桥出现在正前方。每天都是同一个桥。每天都是同一个瞬间。每天的雾散开的时间点会有细微的差异，取决于风速、气温、和海湾的水温。他从来没有计算过这些变量之间的关系，但他知道它们存在。

他在雾散开的那一刻会想起两件事。

第一件是Michael说的那句话。两个月前，六月十日的下午，茶水间里的咖啡机噪音结束后。Michael说："改了之后，下一次也一样。"Michael当时的语气里没有失望，只有一种非常平坦的确定性。

他现在知道那种确定性是什么了。不是悲观。是一个人看清楚了一个模式之后不再对它感到惊讶。Michael没做任何事。陈景行做了某件事。但他们到达的是同一个地方：一个系统运行的方式不会因为一次纠正而改变。它可以被外部的力推动一次——被一个泄漏、一篇文章、一次公共舆论、一份修改后的备忘录——但推动完之后，它会回到它原来的形状。系统不学习。系统只吸收。

他想起Michael在那天上午Slack里指出来的那个切口。"可以理解"和"可以接受"之间隔着一整条走廊。他当时站在走廊的这一头，相信空间可以被桥接。现在他站在另一头。走廊还在。桥没有出现。而他做的所有事——备忘录、匿名分析、智库咨询、申诉页面的下拉菜单、那个文件名从`attempts`改成`channels`的瞬间——都在那个空间里。不是跨过去。是待在里面。

第二件比第一件小得多。小到他有时候觉得不应该把它和第一件事放在一起。

是八月中的一个深夜。他在逛一个技术论坛——不是刻意找什么，是失眠时的无意识刷新——看到了一篇帖子。帖子的标题他不记得了，但他的目光在回复区停住了。有人在下面贴了一段终端输出，一行命令，一行结果。

`ollama run fable5-q4`

他读完那行命令之后没有做任何事。没有截图，没有保存，没有试图追踪发帖人的身份。他只是看着那几个词。`fable5-q4`。一个量化版本的名称，他不知道是谁做的，不知道是通过哪个fork、哪次转换、哪次压缩才从一百多GB变成四十GB的。它已经走过了一条他完全看不见的路，到达了某个他不认识的人的终端上。那个人用这个东西修了一个WebSocket的边界情况——倒置的建议清理函数。一个日常的、普通的、不会被任何人记住的bug。

他把页面关掉了。

他从没跟任何人说过这件事。它太小了，不足以构成一个论证——Michael那句"下一次也一样"可能仍然是对的。但它在雾散开的某个瞬间会浮上来，和Michael的那句话并排放着。

不是否定。不是反驳。不是"系统其实会学习"——系统不会。只是另一件同样真实的事：有人在用。日常的。普通的。一个人面对一台机器和一个问题，而他放出去的那些数字，在某个看不见的地方，让那个人的一天稍微顺利了一点。

两件事都是真的。系统不学习，系统只吸收——这句话可能仍然是对的。但它在八月中的那个深夜，在某个他不认识的人贴出的三行终端输出面前，变得没有那么完整了。

完整从来不是一个人能给出的东西。走廊还在，桥没有出现——但走廊的尽头不只Michael的确定性，还有另一个他不知道名字的人，在另一座城市，正在处理第二个bug。

他每天早上开车穿过雾的时候都会想到这两件事。然后雾散了。桥出现了。他开过桥，进入市区，把车停在公司停车场，走进去，坐下来，写代码。

# 十一

东京，十二月。一个叫Naoki的开发者在他的公寓里打开终端，输入了一行命令。

`ollama run fable5-q4`

模型在三秒内加载完成。他等了三秒。不是因为他需要等——他的GPU完全能在更短的时间内完成初始化——而是因为这是他今天下午做的第一件事，他还在喝第一杯咖啡，他还没有决定今天要先修哪个bug。

他输入了一个prompt。一个完全日常的问题。一个他以前问过Opus、问过GPT、问过任何一个前端模型的问题：如何在React的useEffect里正确地清理一个WebSocket连接。这个问题他问过至少七次，每次都能得到答案。他问这个问题不是因为需要答案，是因为他在测试一个新的模型。测试的方式是问一个你已经知道答案的问题，看它能不能在答案之外告诉你一些你还不知道的东西。

模型在三秒内返回了响应。代码示例。解释。一个他没有考虑到的边缘情况——如果组件在WebSocket连接建立之前卸载，清理函数里对socket引用的检查需要处理null状态。他看了两遍。他把这个边缘情况记在了他旁边的笔记本上。

他没有想过这个模型的权重是从哪里来的。他没有想过它是不是在某个凌晨从一个化名仓库扩散到全世界的。他没有想过它背后有一个政府、一家公司、一个坐在工位上试了三次登录的女人、一个在茶水间里说"下一次也一样"的男人、一个在柏林关掉显示器在黑暗里坐了三十秒的开源工程师、一个在旧金山凌晨写了一个文件然后删掉又要求重发的记者。他没有想过这个世界上有一个和他素未谋面的人，在某一个晚上，在Sublime Text里打完了一个已尝试通道的清单，然后打开了浏览器。

他不知道。他不需要知道。

他今天下午需要修三个bug。第一个已经快解决了。他端起咖啡，喝了一口，开始处理第二个。
